◆ createsBalance()

bool ScheduleManager::createsBalance ( Request  r)

Complexity = O(n²)

Check if a request creates balance between the classes of a Uc.

This is our approach to balance:

| diff < 4 | -> is balanced

If the difference (diff = max - min) between the class that as more students and the class that has less

is less then 4, then all request are allowed unless they involve edges (classes with max or min)

this is what was required in the project statement.

| diff >= 4| -> is not balanced

If the difference (diff = max - min) between the class that as more students and the class that has less

is more or equals than 4, then the request is only allowed if the number of students of the classes involved

are closer to the ideal number of students (ideal = total number of uc's students / nº of classes of the uc)

Parameters
r
Returns
TRUE if a request creates balance between the classes and FALSE if not
